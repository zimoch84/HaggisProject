# Decyzje projektowe (ADR index)

Kazda istotna decyzja architektoniczna powinna miec osobny plik ADR w katalogu `docs/decisions/adr`.

## Statusy

- Proposed
- Accepted
- Superseded
- Deprecated

## Spis ADR

- `ADR-0001` - Architektura backend C# + AsyncAPI WebSocket + Flutter UI (`docs/decisions/adr/ADR-0001-architecture.md`) - Accepted

## Jak dodac nowa decyzje

1. Skopiuj szablon z `docs/decisions/adr/ADR-template.md`.
2. Nazwij plik: `ADR-XXXX-krotki-temat.md`.
3. Uzupelnij sekcje: Context, Decision, Consequences, Alternatives.
4. Dodaj wpis do listy powyzej.
